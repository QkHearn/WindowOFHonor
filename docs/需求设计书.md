## 1. 项目概述

### 1.1 项目背景

「荣耀之窗」是一套面向企业团队的荣誉激励与展示系统。主管可通过系统向员工发放激励，员工可查看个人与团队的荣誉成果、任务令、最佳拍档等信息，并在团队展播中相互点赞，形成正向激励闭环。

### 1.2 建设目标

- **激励透明化**：主管发放激励有据可查，员工可实时感知荣誉变化。
- **荣誉可视化**：通过展播大屏与个人中心，让成就被看见、被传播。
- **协作可感知**：以关系网络呈现「最佳拍档」与共获荣誉，让协作关系一目了然。
- **竞争与对标**：提供个人版、团队版激励排行榜，激发良性竞争与团队荣誉感。
- **高端视觉体验**：全站 UI 对标奢侈品数字官网——克制、留白、材质感与仪式感，让荣誉呈现具有「被珍视」的质感。
- **轻量易部署**：支持在自有服务器一键部署，适配企业内网环境。

### 1.3 适用范围

适用于中小型团队、部门或项目组，用于日常激励发放、荣誉展示与团队文化建设。首期以 Web 端为主，支持大屏展播与 PC/移动端浏览器访问。

## 2. 术语定义

| 术语 | 说明 |
|------|------|
| 主管 | 具备激励发放权限的管理角色，通常为团队负责人 |
| 员工 | 普通使用角色，可查看个人荣誉、团队展播并点赞 |
| 激励 | 主管向员工发放的荣誉类奖励，可附带说明、分值或徽章 |
| 荣誉 | 员工累计获得的激励总量或展示单位（如积分、勋章、次数） |
| 任务令 | 主管或系统分配给员工的任务/挑战，完成后可关联荣誉 |
| 最佳拍档 | 与当前员工共同获得荣誉次数最多的协作对象 |
| 关系网络 | 以图结构展示员工之间共获荣誉的协作关系，节点为人、连线为共获关系 |
| 共获荣誉 | 两名及以上员工在同一激励或任务中共同获得的荣誉记录 |
| 激励排行榜 | 按荣誉总量或激励次数对员工或团队进行排序展示的榜单 |
| 个人版排行榜 | 以员工个人为维度，展示个人荣誉排名及名次变化 |
| 团队版排行榜 | 以团队/部门为维度，展示团队间或团队内成员荣誉排名 |
| 视觉规范 | 全站奢侈品级 UI 风格，详见 §5 与《UI奢侈品风格提示词》 |
| 荣誉展播 | 以轮播/列表形式公开展示近期荣誉动态的展示模块 |
| 团队展播 | 面向团队维度的荣誉汇总与成员成就展示 |
| 点赞 | 员工对他人荣誉或成就的认可互动行为 |

## 4. 功能需求

### 4.1 主管端：发放激励

#### 4.1.1 功能描述

主管选择一名或多名员工，创建并发放激励。激励发放后，相关员工荣誉值更新，并进入荣誉展播队列。

#### 4.1.2 功能要点

| 编号 | 需求项 | 优先级 | 说明 |
|------|--------|--------|------|
| A-01 | 选择接收人 | P0 | 支持单选、多选员工；可按部门/团队筛选 |
| A-02 | 激励类型 | P0 | 支持荣誉积分、勋章、荣誉称号等（可配置） |
| A-03 | 激励说明 | P0 | 必填标题，选填描述（如「Q2 项目攻坚突出贡献」） |
| A-04 | 共获荣誉 | P1 | 支持将激励同时发放给多人，形成共获记录 |
| A-05 | 发放确认 | P0 | 提交前预览接收人列表与激励内容 |
| A-06 | 发放记录 | P0 | 主管可查看历史发放记录，含时间、对象、类型 |
| A-07 | 撤销/更正 | P2 | 限定时长内可撤销误发激励（需留痕） |

#### 4.1.3 业务规则

- 单次发放最多支持 **N 人**（建议默认 20，可配置）。
- 激励发放成功后 **实时** 更新员工荣誉统计。
- 共获荣誉：同一笔激励关联的所有接收人，彼此计入「共获荣誉」关系。
- 所有发放操作记录操作人、时间、IP（可选），满足审计要求。

#### 4.1.4 用户流程

```
主管登录 → 进入「发放激励」→ 选择员工（可多选）→ 填写激励信息
→ 确认预览 → 提交 → 系统通知员工（可选）→ 写入展播队列
```

### 4.2 员工端：荣誉展播

#### 4.2.1 功能描述

以动态流或轮播形式，展示近期获得的激励与荣誉事件，供全员浏览，营造荣誉氛围。适合办公室大屏常亮展示。

#### 4.2.2 功能要点

| 编号 | 需求项 | 优先级 | 说明 |
|------|--------|--------|------|
| E-01 | 展播列表 | P0 | 按时间倒序展示近期荣誉动态 |
| E-02 | 展播卡片 | P0 | 展示：员工头像/姓名、激励标题、时间、荣誉类型 |
| E-03 | 自动轮播 | P0 | 大屏模式支持自动切换，间隔可配置（如 5–10 秒） |
| E-04 | 筛选维度 | P1 | 按团队、时间范围、激励类型筛选 |
| E-05 | 全屏模式 | P0 | 适配电视/投影大屏，无导航栏干扰 |
| E-06 | 实时刷新 | P1 | 新激励发放后自动出现在展播（WebSocket 或轮询） |

#### 4.2.3 展示内容示例

> 🏆 **张三** 获得「月度之星」  
> 表彰其在 XX 项目中的卓越贡献 · 2026-07-08 14:30

### 4.3 员工端：个人中心

个人中心聚合员工与荣誉相关的全部自助查询能力，包含以下子模块。

#### 4.3.1 我的荣誉总览

| 编号 | 需求项 | 优先级 | 说明 |
|------|--------|--------|------|
| P-01 | 荣誉总量 | P0 | 展示累计荣誉值 / 勋章数 / 获奖次数 |
| P-02 | 荣誉明细 | P0 | 分页列表：来源、时间、发放人、说明 |
| P-03 | 荣誉趋势 | P2 | 按月/季度统计图表（二期） |

#### 4.3.2 最佳拍档与关系网络

最佳拍档不仅以列表呈现，还通过**关系网络图**可视化员工之间的共获荣誉协作关系，帮助员工直观感知自己的协作圈子。

| 编号 | 需求项 | 优先级 | 说明 |
|------|--------|--------|------|
| P-04 | 拍档排行 | P0 | 展示与共获荣誉次数最多的 Top N 同事 |
| P-05 | 拍档卡片 | P0 | 显示拍档头像、姓名、共获次数、最近共获时间 |
| P-06 | 拍档详情 | P1 | 点击拍档可查看双方共获荣誉的具体记录列表 |
| P-06a | 关系网络图 | P0 | 以力导向图/关系图展示共获荣誉关系网络 |
| P-06b | 网络节点 | P0 | 节点为员工头像+姓名；节点大小可映射荣誉总量或共获次数 |
| P-06c | 网络连线 | P0 | 连线表示共获关系；线宽/颜色映射共获次数强弱 |
| P-06d | 中心节点 | P0 | 默认以当前登录员工为中心，向外辐射协作关系 |
| P-06e | 交互操作 | P1 | 点击节点查看该员工摘要；点击连线查看共获明细；支持缩放、拖拽 |
| P-06f | 网络筛选 | P1 | 按时间范围（本月/本季/全部）、团队范围筛选关系 |
| P-06g | 拍档高亮 | P0 | 在关系网络中高亮标注 Top 1 最佳拍档节点及连线 |

**关系网络数据规则**：

- **节点**：参与过至少一次共获荣誉的员工；当前用户始终位于图中心。
- **边**：两名员工在同一笔激励中均为接收人，则产生一条边；同一对员工多次共获则合并为一条边，权重累加。
- **最佳拍档**：与当前用户连边权重（共获次数）最大的员工；次数相同取最近共获时间更近者。
- **展示范围**：默认展示与当前用户有直接共获关系的 1 度节点；支持展开至 2 度关系（二期）。

**关系网络示意**：

```
                    [李四]
                   ╱ 共获3次
          [王五] ──────── [我/张三] ──────── [赵六]
                   ╲ 共获5次              ╲ 共获2次
                    [最佳拍档·陈七]         [周八]
```

#### 4.3.3 共获荣誉关系

| 编号 | 需求项 | 优先级 | 说明 |
|------|--------|--------|------|
| P-07 | 共获列表 | P0 | 展示与哪些人共获过荣誉，及各自共获次数 |
| P-08 | 共获明细 | P1 | 每次共获的激励名称、参与人、时间 |

#### 4.3.4 我的任务令

| 编号 | 需求项 | 优先级 | 说明 |
|------|--------|--------|------|

| P-10 | 任务详情 | P0 | 任务描述、截止时间、关联荣誉（若有） |

### 4.4 激励排行榜

激励排行榜分为**个人版**与**团队版**两个维度，支持不同范围、周期下的荣誉排名展示，可用于个人中心、团队展播及大屏展播场景。

#### 4.4.1 个人版激励排行榜

以**员工个人**为排名单位，展示个人荣誉成绩在选定范围内的位次。

| 编号 | 需求项 | 优先级 | 说明 |
|------|--------|--------|------|
| R-01 | 排名列表 | P0 | 展示 Top N 员工：名次、头像、姓名、荣誉总量、获激励次数 |
| R-02 | 我的名次 | P0 | 固定展示当前用户排名；未进 Top N 时显示「我的排名：第 X 名」 |
| R-03 | 排名范围 | P0 | 支持切换：本团队 / 全公司（或全系统） |
| R-04 | 排名周期 | P0 | 支持切换：本周 / 本月 / 本季度 / 本年度 / 累计 |
| R-05 | 排名维度 | P1 | 按荣誉积分总量（默认）或获激励次数排序，可切换 |
| R-06 | 名次变化 | P1 | 展示较上一周期名次升降（↑↓ 及变化位数） |
| R-07 | 前三名特效 | P1 | 金/银/铜视觉标识，适配大屏展示 |
| R-08 | 跳转个人 | P1 | 点击榜单员工可查看其公开荣誉摘要（非隐私字段） |

**个人版业务规则**：

- 默认排序：荣誉积分总量降序；相同积分按最近获激励时间降序。
- 员工仅可查看排行榜，不可修改排名数据。
- 离职/禁用账号不出现在榜单，历史数据保留但不参与排名。

#### 4.4.2 团队版激励排行榜

以**团队/部门**为排名单位，或展示**团队内部成员**排名，适用于团队间对标与团队内激励。

| 编号 | 需求项 | 优先级 | 说明 |
|------|--------|--------|------|
| R-09 | 团队间排行 | P0 | 各团队按荣誉总量（成员合计）排名 |
| R-10 | 团队内排行 | P0 | 选定团队后，展示该团队成员个人排名（同个人版字段） |
| R-11 | 团队摘要 | P0 | 团队名称、成员数、荣誉总量、本月新增、人均荣誉 |
| R-12 | 排名范围 | P0 | 团队间排行默认全公司；团队内排行限定所选团队 |
| R-13 | 排名周期 | P0 | 同个人版：本周 / 本月 / 本季度 / 本年度 / 累计 |
| R-14 | 团队标识 | P1 | 展示团队负责人、团队徽章（可配置） |
| R-15 | 大屏模式 | P0 | 团队版排行榜支持全屏轮播，可与荣誉展播组合展示 |

**团队版业务规则**：

- **团队间排名** = 该团队所有在职成员在统计周期内的荣誉积分之和。
- **人均荣誉** = 团队荣誉总量 ÷ 在职成员数，可作为辅助排序维度（二期）。
- 员工默认看到**本团队内排行** + **全公司团队间排行**；主管额外可看所辖团队详情。

#### 4.4.3 排行榜展示场景

| 场景 | 版本 | 说明 |
|------|------|------|
| 个人中心 | 个人版 | 员工查看自己的名次及前后几名 |
| 团队展播 | 团队版（团队内） | 本团队成员排行，嵌入团队展播页 |
| 荣誉展播大屏 | 个人版 + 团队版 | 轮播切换：近期荣誉动态 ↔ 个人 Top 10 ↔ 团队 Top 5 |
| 主管工作台 | 双版本 | 查看所辖团队内外排名，辅助激励决策 |

#### 4.4.4 排行榜示意

## 5. UI 设计规范（奢侈品级）

> 母版提示词详见：[UI奢侈品风格提示词.md](./UI奢侈品风格提示词.md)  
> 气质对标：Hermès、Bottega Veneta、Aman Resorts、Patek Philippe 数字体验——**静谧、尊贵、克制、仪式感**。

### 5.1 设计原则

| 原则 | 说明 |
|------|------|
| 少即是多 | 一屏一个视觉焦点；拒绝信息堆砌与廉价 SaaS 卡片堆叠 |
| 荣誉即仪式 | 激励发放、展播揭晓、上榜等关键时刻需有庄重动效，像呈献证书而非弹通知 |
| 材质胜于装饰 | 用纸张噪点、毛玻璃、金箔描边、弥散阴影营造质感，而非贴图和渐变 |
| 编辑式排版 | 大标题 + 细线分割 + 精炼文案；像杂志内页，不像后台表格 |
| 动静有致 | 动效缓慢优雅（400–600ms）；禁止弹跳、抖动、霓虹闪烁 |

### 5.2 色彩体系

| 角色 | 色值 | 用途 |
|------|------|------|
| 墨夜黑 | `#0A0A0A` / `#121212` | 大屏展播、登录页、导航深色模式 |
| 象牙白 | `#F7F3ED` | 浅色内容区主背景 |
| 暖米 | `#EDE6DA` | 卡片、次级背景 |
| 香槟金 | `#C9A962` | 荣誉元素、按钮、描边、强调数字（面积 < 8%） |
| 古铜金 | `#A68B4B` | 次要金色、hover 态 |
| 石墨灰 | `#6B6B6B` | 辅助说明文字 |

**禁止**：高饱和蓝紫、荧光色、Material/Ant Design 默认色板、彩虹渐变。

### 5.3 字体与排版

| 层级 | 字体建议 | 规格 |
|------|----------|------|
| 品牌/页面标题 | Cormorant Garamond / 思源宋体 SemiBold | 32–48px，字距 0.15–0.2em |
| 模块标题 | 同上 | 24–28px |
| 正文 | Inter / 思源黑体 Light | 14–16px，行高 1.6–1.8 |
| 荣誉数字 | 衬线或等宽 | 36–64px 金色，表盘读数感 |
| 英文品牌 | WINDOW OF HONOR | 全大写 + 宽字距 |

### 5.4 核心组件风格

| 组件 | 规范 |
|------|------|
| 按钮 | 主：深底金字或金边透明底；圆角 2–4px；文案用「呈递荣誉」「进入」等仪式感用语 |
| 输入框 | 底线式，聚焦时金色底线从左向右延展 |
| 卡片 | 毛玻璃 `backdrop-blur` + 1px 香槟金半透明描边 + 柔和弥散阴影 |
| 头像 | 圆形 + 1px 金圈；大屏展播尺寸 ≥ 96px |
| 排行榜行 | 无竖线，行高 56–72px；前三名左侧金/银/铜 2px 竖线 |
| 图标 | 极细线稿 1–1.5px，金色或石墨灰；禁用填充卡通图标 |
| 勋章 | 哑光金属浮雕 SVG，非 emoji 贴纸 |

### 5.5 分页面视觉要求

#### 5.5.1 登录页

- 全屏墨夜黑底，中央偏上品牌字标「荣耀之窗 / WINDOW OF HONOR」
- 底部象牙色毛玻璃登录卡，仅底线输入框 + 金色细边按钮
- 背景极淡金色粒子缓慢漂浮（低密度，不喧宾夺主）

#### 5.5.2 荣誉展播（大屏）

- 深色沉浸背景，单条荣誉居中——肖像金框 + 衬线称号 + 金色细线 + 灰色说明
- 每帧 ≤ 5 个视觉元素，大量留白；像高端酒店艺术屏，非电视台字幕条
- 轮播 crossfade 8–12 秒/帧；新荣誉揭晓时金色微光扫过（shimmer 1.2–2s）

#### 5.5.3 激励排行榜

- 象牙纸色底，名录式排版（Forbes 榜单 × Hermès 画册）
- 名次用细衬线数字；荣誉积分金色大号右对齐
- 前三名行背景极淡金箔纹理；「我的名次」固定底栏，毛玻璃浮层

#### 5.5.4 关系网络（最佳拍档）

- 深色底金色星座图：节点圆形金框人像，连线为极细香槟光丝
- 线宽映射共获次数；中心节点（当前用户）略大 + 柔光晕
- 像腕表背透机芯或星空图；禁止科技蓝网格底图
- 列表/网络切换用下划线 tab，过渡 500ms 淡入

#### 5.5.5 主管发放激励

- 左侧贵宾名册式员工列表，右侧极简表单
- 提交按钮文案「呈递荣誉」；确认页像签署荣誉证书预览

#### 5.5.6 个人中心 / 团队展播

- 浅色编辑式布局；区块间距 80–120px
- 荣誉明细为时间轴式，金色节点标记关键激励
- 点赞为细线心形图标，激活时金色填充 + 轻微光晕，无爆炸动画

### 5.6 动效规范

| 场景 | 动效 | 参数 |
|------|------|------|
| 页面进入 | 淡入 + 轻微上移 | 8–16px，错峰 80ms |
| 卡片悬停 | 亮度微升或金边显现 | 300ms |
| 荣誉揭晓 | 金色 shimmer 扫过 | 1.2–2s |
| 轮播切换 | crossfade | 800ms |
| 加载 | 金色细线描边动画 | 禁止默认转圈 |

### 5.7 UI 验收标准

| 编号 | 验收项 | 通过条件 |
|------|--------|----------|
| UI-01 | 色彩合规 | 无高饱和默认色；金色点缀面积克制 |
| UI-02 | 字体层级 | 标题衬线 + 正文无衬线，层级清晰 |
| UI-03 | 大屏展播 | 单帧简洁，5 元素以内，动效流畅不廉价 |
| UI-04 | 排行榜 | 名录式排版，前三名有质感区分 |
| UI-05 | 关系网络 | 深色金线星座风格，无科技蓝网格 |

## 8. 核心数据模型（概念）

### 8.1 实体关系概览

```
用户(User) ──归属──▶ 部门/团队(Department)
    │
    ├── 接收 ──▶ 激励记录(IncentiveRecord)
    │                 │
    │                 └── 关联 ──▶ 激励类型(IncentiveType)
    │
    ├── 拥有 ──▶ 荣誉统计(HonorStats)
    │
    ├── 分配 ──▶ 任务令(TaskOrder)
    │
    ├── 共获关系 ──▶ 协作关系边(CoHonorEdge)  ──▶ 关系网络
    │
    └── 发起/接收 ──▶ 点赞(Like)
```

### 8.2 主要实体字段（草案）

#### 用户（users）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| username | string | 登录名 |
| display_name | string | 显示姓名 |
| avatar_url | string | 头像 |
| role | enum | super_admin / supervisor / employee |
| department_id | FK | 所属部门 |
| status | enum | active / disabled |

#### 激励记录（incentive_records）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| title | string | 激励标题 |
| description | text | 说明 |
| type_id | FK | 激励类型 |
| honor_value | int | 荣誉分值 |
| issued_by | FK | 发放人（主管） |
| issued_at | datetime | 发放时间 |

#### 激励接收人（incentive_recipients）

| 字段 | 类型 | 说明 |
|------|------|------|
| incentive_id | FK | 激励记录 |
| user_id | FK | 接收员工 |

> 多人共获：同一 `incentive_id` 对应多条 `incentive_recipients`，用于计算共获关系。

#### 任务令（task_orders）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| title | string | 任务标题 |
| description | text | 任务描述 |
| assignee_id | FK | 执行人 |
| assigned_by | FK | 分配人 |
| status | enum | pending / in_progress / completed / expired |
| due_at | datetime | 截止时间 |
| linked_incentive_id | FK | 完成后关联激励（可选） |

#### 点赞（likes）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| from_user_id | FK | 点赞人 |
| to_user_id | FK | 被赞人 |
| target_type | enum | honor_event / user_profile |
| target_id | UUID | 关联目标 ID |
| created_at | datetime | 点赞时间 |

#### 共获关系边（co_honor_edges）— 关系网络与拍档计算

| 字段 | 类型 | 说明 |
|------|------|------|
| user_a_id | FK | 员工 A（存较小 UUID，保证唯一对） |
| user_b_id | FK | 员工 B |
| co_count | int | 累计共获次数 |
| last_co_at | datetime | 最近共获时间 |
| total_honor_value | int | 共获累计荣誉分值 |

> 每次多人共获激励发放时，对接收人两两更新 `co_honor_edges`，供关系网络与最佳拍档查询。

#### 排行榜快照（leaderboard_snapshots）— 可选，用于名次变化

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| scope | enum | personal / team |
| range_type | enum | team / company |
| period | enum | week / month / quarter / year / all |
| entity_id | UUID | 用户 ID 或团队 ID |
| rank | int | 名次 |
| honor_value | int | 荣誉值 |
| snapshot_at | datetime | 快照时间 |

## 14. 附录：用户故事（User Stories）

### 主管

> 作为团队主管，我希望一次性向多名表现突出的员工发放激励，以便公开表彰并计入他们的荣誉档案。

> 作为团队主管，我希望查看历史发放记录，以便复盘激励情况。
