## 1. 选型结论（Executive Summary）

| 层级 | 选定方案 | 版本 |
|------|----------|------|
| 前端 | **React + Vite + TypeScript + Tailwind CSS** | React 19 / Vite 6 |
| 动效 | Framer Motion | 11.x |
| 图表/关系网络 | ECharts + echarts-for-react | 5.x |
| 后端 API | **NestJS + TypeScript** | 11.x |
| ORM | **Prisma** | 6.x |
| 数据库 | **PostgreSQL** | 16 |
| 缓存 / 实时 | **Redis**（展播推送、排行榜缓存） | 7.x |
| 反向代理 | **Nginx** | 1.27 |
| MCP 服务 | **@modelcontextprotocol/sdk**（Streamable HTTP + stdio） | 1.x |
| 容器编排 | **Docker Compose** | Compose v2 |
| 运行时 | **Node.js** | 22 LTS |

**选型原则**：TypeScript 全栈统一语言、模块化单体（Modular Monolith）、容器化一键部署、MCP 作为 AI 接入层与业务 API 解耦。

## 3. 各层选型理由

### 3.1 前端：React + Vite + Tailwind

| 候选 | 结论 |
|------|------|
| React vs Vue | 选 **React**——奢侈品风自定义组件多、Framer Motion 生态成熟、大屏与关系图 React 绑定丰富 |
| Vite vs Next.js | 选 **Vite**——内网私有化部署、无 SSR 刚需；构建快、镜像小 |
| Tailwind | 设计 Token（§5 色彩/间距）可直接写入 `tailwind.config`，避免 Ant Design 默认风格 |

### 3.2 后端：NestJS + Prisma + PostgreSQL

| 候选 | 结论 |
|------|------|
| NestJS vs FastAPI vs Go | 选 **NestJS**——与前端同语言；模块化契合「激励/排行榜/关系网络」分域；守卫/拦截器便于 RBAC |
| Prisma vs TypeORM | 选 **Prisma**——Schema 即文档、迁移清晰、TypeScript 类型安全 |
| PostgreSQL vs MySQL | 选 **PostgreSQL**——JSON 字段（关系网络缓存）、窗口函数（排行榜）、扩展性好 |

### 3.3 缓存：Redis

- 荣誉展播 Feed 热点缓存（TTL 30s）
- 排行榜快照与名次变化对比
- SSE/WebSocket 发布订阅（展播实时刷新）

### 3.4 MCP 服务：独立 Sidecar

| 决策 | 说明 |
|------|------|
| 独立进程 | MCP 与 API 解耦，AI 工具变更不影响核心业务 |
| Streamable HTTP | 容器暴露 `/mcp` 端点，远程大模型可调用 |
| stdio 双模式 | 本地 Cursor 开发时 `docker compose run mcp --stdio` 或本地 node 启动 |
| 鉴权 | MCP 层 API Key + 后端 Service Token；工具按角色过滤 |

## 4. 容器镜像规划

| 镜像名 | Dockerfile | 基础镜像 | 说明 |
|--------|------------|----------|------|
| `windowofhonor-api` | `apps/api/Dockerfile` | `node:22-alpine` | 多阶段构建，仅含 dist + prisma |

## 5. MCP 接口设计

### 5.1 传输协议

| 模式 | 地址 | 场景 |
|------|------|------|
| Streamable HTTP | `http://<host>/mcp` | 生产部署、远程 Agent |
| stdio | 进程 stdin/stdout | Cursor 本地 MCP 配置 |

### 5.2 Tools（工具）

| 工具名 | 说明 | 权限 |
|--------|------|------|
| `honor_issue_incentive` | 主管向员工发放激励 | supervisor+ |
| `honor_get_broadcast` | 获取荣誉展播动态 | all |
| `honor_get_leaderboard` | 个人/团队排行榜 | all |
| `honor_get_user_honors` | 查询用户荣誉摘要与明细 | all |
| `honor_get_partners` | 最佳拍档与共获关系网络 | all |
| `honor_get_tasks` | 查询任务令 | all |
| `honor_like_user` | 为员工点赞 | employee+ |
| `honor_search_users` | 搜索员工（发放激励用） | supervisor+ |

### 5.3 Resources（资源）

| URI | 说明 |
|-----|------|
| `honor://broadcast/latest` | 最新展播条目 JSON |
| `honor://leaderboard/personal` | 个人排行榜快照 |
| `honor://leaderboard/team` | 团队排行榜快照 |

### 5.4 Prompts（提示模板）

| 名称 | 说明 |
|------|------|
| `honor_weekly_summary` | 生成本周团队荣誉周报 |
| `honor_praise_message` | 根据激励记录生成表彰文案 |

### 5.5 Cursor MCP 配置示例

```json
{
  "mcpServers": {
    "window-of-honor": {
      "url": "http://localhost:3100/mcp",
      "headers": {
        "Authorization": "Bearer <MCP_API_KEY>"
      }
    }
  }
}
```

## 9. 安全策略

- API：JWT + RBAC（super_admin / supervisor / employee）
- MCP：Bearer Token 鉴权；工具层校验角色
- 容器：非 root 用户运行 Node 进程
- 网络：DB/Redis 不映射宿主机端口
- 密钥：`.env` 不入库，生产用 Docker secrets 或 K8s Secret
