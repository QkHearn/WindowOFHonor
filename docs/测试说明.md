# 测试说明

## 测试分层

| 层级 | 位置 | 命令 | 说明 |
|------|------|------|------|
| 单元测试 | `apps/api/src/**/*.spec.ts` | `npm run test:unit` | Mock Prisma，覆盖核心业务逻辑 |
| 集成测试 | `apps/api/test/app.e2e-spec.ts` | `npm run test:e2e` | Supertest + 真实 PostgreSQL |
| MCP 测试 | `apps/mcp/src/**/*.test.ts` | `npm test` | API Client 请求与鉴权 |
| 前端测试 | `apps/web/src/**/*.test.tsx` | `npm test` | Vitest + Testing Library |
| 系统冒烟 | `scripts/system-smoke-test.mjs` | `npm run test:smoke` | 对已部署 Docker 栈做端到端验证 |

## 一键运行全部测试

```bash
# 启动测试数据库 + 单元/集成/MCP/Web 测试
npm test
# 或
bash scripts/run-all-tests.sh
```

`run-all-tests.sh` 会：
1. `docker compose -f docker-compose.test.yml up` 启动测试库（端口 5433）
2. 依次运行 API 单元、API E2E、MCP、Web 测试

## 单独运行

```bash
# API 单元测试（无需数据库）
cd apps/api && npm run test:unit

# API E2E（需测试库）
docker compose -f docker-compose.test.yml up -d --wait
export DATABASE_URL=postgresql://honor_test:test_password@localhost:5433/windowofhonor_test
export JWT_SECRET=test-jwt-secret-min-32-characters-long
export MCP_SERVICE_TOKEN=test-mcp-service-token
cd apps/api && npm run test:e2e

# 覆盖率
cd apps/api && npm run test:cov
```

## 系统冒烟（部署后）

```bash
docker compose up -d --build
docker compose exec api npx prisma db seed

BASE_URL=http://localhost:8080 MCP_URL=http://localhost:3100 npm run test:smoke
```

验证项：API/MCP 健康检查、首页、登录、展播、排行榜、用户搜索、MCP 端点可达。

## CI

GitHub Actions 工作流 `.github/workflows/ci.yml` 在 push/PR 时自动运行全部测试并构建。

## E2E 覆盖场景

- 登录鉴权（成功/失败）
- 主管发放激励（多人共获）
- 荣誉展播、个人/团队排行榜
- 最佳拍档与共获关系网络
- 点赞（含不可自赞）
- 用户搜索
- MCP Service Token 访问
